📋 Legend & Relationships

PK (Primary Key) Unique identifier for each record in the table
FK (Foreign Key) References primary key from another table
||--o{ (One to Many) One patient has many appointments
Cardinality Shows relationship multiplicity between entities

🔑 Key Relationships Summary

  • Patient → Appointment: One patient can have multiple appointments
  • Patient → Admission: One patient can have multiple admissions
  • Doctor → Appointment: One doctor can conduct multiple appointments
  • Doctor → Operation: One doctor can perform multiple operations
  • Admission → Ward: Each admission is assigned to a ward
  • Admission → ICU: Admission can require ICU care
  • Admission → Operation: Admission can include surgical operations
  • Admission → Billing: Each admission generates a billing record
  • Ward → Bed Tracking: Ward tracks available and total beds
  • Operation → Theatre: Operations are scheduled in operation theatres
  • Patient → Disease: Patient can suffer from multiple diseases
  • Patient → Lab_Test: Patient can undergo multiple lab tests
